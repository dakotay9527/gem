# 相册拾光双平台下载按钮与安装区位置设计

## 目标

将首页首屏的 Android 与 iPhone/iPad 下载入口统一为同规格按钮，并把完整的 Android 安装说明、扫码下载和校验信息移动到网页底部的隐私政策正上方。

## 首屏下载入口

- 使用一个两列的下载按钮容器，桌面端并排展示，窄屏端上下排列。
- 两个按钮共享相同高度、圆角、内边距、图标与双行文字结构。
- Android 按钮保留暖橙渐变，文字为“扫码下载 / Android 版”，点击后跳转至页面底部的 Android 扫码区。
- iPhone/iPad 按钮使用深色薄荷渐变，文字为“App Store 下载 / iPhone / iPad”，点击后打开正式 App Store 页面。
- 每个按钮在内部展示自己的系统和版本信息：
  - Android 10–16 · v1.0.0 · 38 MB
  - iOS 17+ · v1.1
- 保留明确的辅助技术标签和键盘焦点状态。

## 页面顺序

主内容顺序调整为：

1. 首屏
2. 产品介绍
3. 核心功能
4. 使用流程
5. 技术支持
6. Android 安装说明、扫码下载、SHA-256 校验
7. 隐私政策

安装区内部结构与 APK、二维码、校验文件链接保持不变，只调整在页面中的位置。

## 响应式行为

- 宽屏使用两列等宽按钮。
- 640px 及以下切换为单列按钮，按钮填满可用宽度。
- 安装区原有三步说明、二维码卡片和校验面板继续沿用现有移动端布局。
- Android 锚点保留滚动顶部间距，避免跳转后内容贴住视口顶部。

## 验收

- HTML 中存在同一容器下的 Android 与 iOS 两个下载按钮。
- Android 按钮指向 `#android-download`，iOS 按钮指向正式 App Store URL。
- Android 安装区在技术支持之后、隐私政策之前，且只出现一次。
- 桌面端按钮并排，手机端按钮上下排列且无横向溢出。
- 原有 APK 哈希、二维码目标和唯一直接下载链接测试继续通过。
