# 相册拾光官网 Android 扫码下载设计

## 目标

在现有相册拾光官网中增加 Android APK 二维码，让电脑端访问者可以直接用手机扫码下载，同时保持手机端现有的一键下载体验。

## 方案

采用“安装说明区内嵌静态二维码卡片”方案。二维码编码以下官方 APK 绝对地址：

`https://dakotay9527.github.io/gem/downloads/photo-curator-android-1.0.0.apk`

二维码图片随网站文件一起托管，不使用第三方二维码生成或跳转服务。这样扫码不会泄露访问数据给额外服务，下载目标也始终属于相册拾光官网。

未采用首屏大二维码，因为会挤压现有产品主视觉；未采用弹窗二维码，因为会引入不必要的 JavaScript 和额外操作。

## 页面与响应式设计

- 在“安卓安装说明”三个步骤之后、SHA-256 校验卡片之前增加扫码下载卡片。
- 桌面端显示白底二维码、标题“手机扫码下载”、Android v1.0.0 与 38 MB 信息，以及可点击的备用下载链接。
- 二维码周围保留足够白色静区，保证手机相机识别。
- 窄屏下卡片改为单列；二维码仍显示，但主要操作继续由现有“官网下载 Android 版”按钮承担。
- 图片提供明确的替代文本，链接提供包含版本信息的无障碍标签。

## 文件与测试

- `assets/android-download-qr.png`：静态二维码图片。
- `index.html`：扫码下载卡片和备用链接。
- `styles.css`：二维码卡片、图片静区和响应式布局。
- `tests/verify_site.py`：验证二维码图片存在、页面链接指向官方 APK、样式存在，并使用 OpenCV 解码二维码确认目标地址。

## 验收标准

- 二维码可被程序解码为官方 APK 完整地址。
- 页面仍只有一个带 `download` 属性的 APK 主下载按钮，避免破坏现有发布契约。
- 桌面和移动页面无横向滚动。
- 静态发布测试全部通过。
- 发布后官网、二维码图片和 APK 地址均返回 HTTP 200。
