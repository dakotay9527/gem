# 相册拾光官网 APP 风格改版设计

## 目标

将官网从深色科技宣传页改造成与相册拾光正式 APP 一致的暖色、纸张感产品页面。所有展示 APP 界面的视觉都必须来自真实运行截图或已发布的 App Store 实机效果图，不再使用 CSS 绘制或虚构的手机界面。

## 品牌基准

官网直接复用 iOS 与 Android APP 共用的设计系统：

- 奶油背景：`#FFF6E8`
- 纸张卡片：`#FFFDF7`
- 深色文字：`#2B2621`
- 珊瑚橙：`#F36F45`
- 薄荷绿：`#8BD6C5`
- 金色：`#F8C96B`
- 玫瑰红：`#E66A70`
- 大卡片圆角：28px
- 普通卡片圆角：24px
- 控件圆角：18px

标题使用系统无衬线字体与较重字重，正文保持舒适行距。阴影使用低透明度暖棕色，不使用当前官网的霓虹网格和暗色玻璃拟态。

## 真实素材

新增网页素材目录 `assets/app/`，只存放来自正式项目的真实素材：

- `app-icon.png`：由正式 `AppIcon-1024.png` 缩放得到。
- `confirm-results.jpg`：由 App Store 已发布的 `01-confirm-results.png` 优化得到。
- `range-score.jpg`：由 App Store 已发布的 `02-range-score.png` 优化得到。
- `analysis-progress.jpg`：由 App Store 已发布的 `03-analysis-progress.png` 优化得到。
- `card-review.jpg`：由 App Store 已发布的 `04-card-review.png` 优化得到。
- `android-range.png`：从 Android API 36 模拟器中安装正式 APK 后实际运行并截取首页。

不得使用 UI mockup、AI 生成图片或 CSS 拼接的 APP 界面。网页可以用 CSS 对真实截图做圆角、阴影、轻微旋转和层叠，但不能修改截图中的产品 UI。

App Store 宣传图缩放到适合网页的尺寸，并以高质量 JPEG 输出；Android 截图保留 PNG。首屏图使用 `fetchpriority="high"`，画廊图片使用 `loading="lazy"` 与 `decoding="async"`。

## 页面结构

### 顶部导航

- 使用真实 APP 图标和“相册拾光”文字。
- 导航背景采用半透明奶油纸张色与轻微模糊。
- 保留下载、功能、支持、隐私政策锚点，并新增“实机界面”锚点。

### 首屏

- 左侧：品牌标签、主标题、产品一句话说明、隐私承诺、Android 与 iPhone/iPad 下载按钮。
- 右侧：以真实 `card-review.jpg` 为主视觉，并叠放真实 `android-range.png` 小图，明确展示两个平台的实际运行效果。
- 使用柔和珊瑚、薄荷、金色圆形作为纯装饰，不出现虚构照片卡片或模拟手机壳。

### 本机隐私承诺

紧接首屏展示三个简洁承诺卡片：

- 照片只在本机分析
- 不需要账号
- 不包含广告与跟踪 SDK

### 实机界面画廊

- 新增 `#experience` 区块，展示四张 App Store 实机效果图。
- 每张图片配一个简短说明：选择范围、分析进度、卡片确认、结果分流。
- 桌面端为 2×2 网格；手机端为可横向滚动并带滚动吸附的卡片。
- 画廊只展示真实截图，不再重复制作网页内的假界面。

### 功能、流程、下载与政策

- 保留现有功能、流程、技术支持、Android 安装说明、二维码、校验值和隐私政策内容。
- 功能和流程改为 APP 同款纸张卡片、暖色编号与胶囊标签。
- 双平台下载按钮继续保持同规格；Android 跳转二维码，iPhone/iPad 进入正式 App Store。
- Android 安装区继续位于隐私政策正上方。

## 响应式设计

- 1180px 以上：首屏两列，截图画廊 2×2。
- 860px 以下：首屏单列，真实设备图位于下载按钮之后。
- 640px 以下：下载按钮上下排列，实机画廊横向滚动，所有卡片控制在视口内。
- 页面任何尺寸都不能产生全局横向滚动；仅实机画廊自身允许横向滑动。

## 可访问性

- 每张实机图使用具体的中文 `alt` 文本说明所展示的 APP 页面。
- 装饰圆形使用 `aria-hidden="true"`。
- 下载按钮、导航和链接保留清晰的键盘焦点状态。
- 正文与背景满足可读对比度，浅色卡片不使用低对比度灰字。

## 验收标准

- HTML 使用真实 `assets/app/app-icon.png` 和至少五张真实 APP 截图。
- 不再存在 `.phone-stage`、`.phone-screen`、`.mock-photo`、`.photo-wall` 或 `.tile` 等虚构界面结构。
- CSS 品牌变量与 APP 设计系统完全一致。
- Android/iOS 下载链接、APK、二维码和 SHA-256 合约保持不变。
- Android 安装区仍在技术支持之后、隐私政策之前。
- 桌面端、平板端和 390px 手机端均无全局横向溢出。
- 自动测试、二维码解码和桌面/手机浏览器视觉验收全部通过后发布。
